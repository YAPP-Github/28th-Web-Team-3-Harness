# 미션 생성 및 확정 정책

- ID: POLICY-MISSION-002
- 상태: active
- 적용 범위: 추천 미션의 후보 생성, 초안 확인, 선택 확정

## 규칙

- 미션 생성은 생성 선행 조건을 만족하고 유효한 미션 설문이 있는 사용자만 요청할 수 있다.
- 같은 사용자의 진행 중인 미션 생성 작업은 하나만 유지한다. 진행 중인 작업이 있으면 새 작업을 만들지 않고 기존 작업을 반환한다.
- 생성 작업은 대기, 실행 중, 성공, 실패 상태로 관리한다.
- 성공한 작업의 미션 초안은 제한된 기간 동안만 확인·확정할 수 있다.
- 사용자는 같은 생성 작업에서 1개 이상 4개 이하의 서로 다른 초안을 선택해 추천 미션으로 확정한다.
- 동일한 초안 집합을 다시 확정하면 처음 확정한 결과를 그대로 반환한다. 다른 초안 집합으로 다시 확정하려 하면 거절한다.

## 불변조건

- 미션 초안은 자신을 만든 미션 생성 작업에만 속한다.
- 추천 미션은 사용자가 소유한 성공·미만료 생성 작업의 초안에서만 만들어진다.
- 하나의 초안은 하나의 추천 미션으로만 확정할 수 있다.
- 미션 초안 선별은 설문의 선택·제외 조건과 최근 미션 결과를 반영하되, 구체적인 추천 방식은 이 정책의 범위에 포함하지 않는다.

## 예외

- 설문 또는 생성 선행 조건이 없으면 생성 작업을 시작할 수 없다.
- 작업이 아직 끝나지 않았거나 실패했거나 초안 유효기간이 지났으면 초안을 조회하거나 확정할 수 없다.
- 존재하지 않거나 다른 사용자의 작업·초안을 확정하려 하면 거절한다.
